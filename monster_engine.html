Monster Engine

A custom C++ game engine built from scratch — featuring layered architecture, Entity Component System, and modern OpenGL rendering.

Overview

Monster Engine is a game engine designed as a deep exploration of engine architecture and systems programming. Built entirely in C++ (92.6% of the codebase), it serves as both a functional engine and a learning platform for understanding how high-performance game engines are structured from the ground up.

The engine features a modular, layered architecture that makes it easy to extend with new game logic or tooling layers, along with high-performance entity management powered by an ECS framework.

Key Features

  • Layered Architecture — Self-contained modules updated and rendered in a stack, easy to extend with new layers for game logic or tools
  • Entity Component System — Powered by entt for high-performance entity management and data-oriented design
  • Modern OpenGL Renderer — Shader support, materials system, and shadow mapping
  • Event System — Robust event bus for decoupling systems and enabling clean communication between engine modules
  • ImGui Integration — Built-in debug UI for real-time inspection and development tooling

Architecture

The engine is built on a few core concepts that mirror industry-standard engine design:

Application manages the window and main loop. Layers are self-contained modules updated in a stack. Scenes contain all entities and components. Systems operate on entities with specific component signatures.

This separation allows each part of the engine to evolve independently — new rendering features, physics systems, or gameplay tools can be added as layers without modifying the core.
